# Changelog

## 2026-05-27 (v0.7.1)
- **[RU]** ИИ-чат: эффект печатания для ответов (символы появляются постепенно). Ответы рендерятся в Markdown — жирный, курсив, списки, заголовки. Чат теперь работает без заметки: разговор можно начать свободно, прикрепить заметку опционально. Кнопка «Отправить в ИИ» в финиш-модале сессии. Новая Cloud Function `getAILimit` — счётчик дневного лимита синхронизируется с сервером вместо localStorage. AIPanel в редакторе проверяет лимит до отправки запроса. Описания персон перенесены в i18n (RU + EN). Аналитика PostHog включается только при явном согласии через `localStorage('analytics_consent')`; добавлены `optIn`/`optOut` методы. Проверка роли admin через ID Token Claims (а не Firestore profile) — защита от подмены. Фикс race condition в `editWithAI`: транзакция Firestore вместо последовательных reads. `enforceAppCheck: true` включён на `editWithAI`, история сообщений санируется перед отправкой в Gemini. Смена пароля: при ошибке обновления в Firebase Auth ключи шифрования автоматически откатываются в Firestore. Service Worker: раздельный stale-while-revalidate кеш для навигационных запросов и кеш-first для ассетов. CSP усилен: `object-src 'none'`, `base-uri 'self'`, `form-action 'self'`; добавлены заголовки `COOP: same-origin` и `CORP: same-origin`.
- **[EN]** AI chat: typewriter effect for responses (characters appear gradually). Responses rendered as Markdown — bold, italic, lists, headings. Chat now works without a note: start freely, attach optionally. "Send to AI" button in session finish modal. New `getAILimit` Cloud Function — daily limit counter synced from server instead of localStorage. AIPanel in editor checks limit before sending. Persona descriptions moved to i18n (RU + EN). PostHog analytics only activates on explicit `localStorage('analytics_consent')` consent; `optIn`/`optOut` methods added. Admin role verified via ID Token Claims (not Firestore profile) — prevents spoofing. Fixed race condition in `editWithAI`: Firestore transaction instead of sequential reads. `enforceAppCheck: true` on `editWithAI`, message history sanitised before sending to Gemini. Password change: encryption keys rolled back in Firestore if Firebase Auth update fails. Service Worker: stale-while-revalidate for navigation, cache-first for assets. CSP hardened: `object-src 'none'`, `base-uri 'self'`, `form-action 'self'`; added `COOP: same-origin` and `CORP: same-origin` headers.
