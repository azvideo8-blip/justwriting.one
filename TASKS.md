- [ ] **PROF-8: Исправление бага мульти-назначения доменов (Minority Domains)** - исправить логику распределения чанков по опорным доменам
  - **Файл:** `src/features/ai/services/AIProfileFacetService.ts`
  - **Логика:** В методе `build()` (строки 107-144) исправить блок `for (const ch of chunks)`. Сейчас, если лучший по сходству домен (`best`) имеет высокий порог (например, «Самореализация» = `0.55`) и отклоняет чанк со сходством `0.50`, чанк уходит в leftover и не распределяется по остальным доменам. Второстепенный домен (например, «Коллеги» = `0.42`) штрафуется прибавкой `SECONDARY_BUMP` (0.03) до порога `0.45` и тоже отклоняет чанк (так как `0.43 < 0.45`), хотя базовый порог `0.42` чанк бы прошел.
  - **Решение:**
    1. Проверить, прошел ли `best` свой порог: `const bestPassed = best.sim >= best.threshold;`.
    2. Если `bestPassed` равен `true`, зафиксировать назначение и оценивать вторичные домены `s` с прибавкой: `const threshold = s.threshold + SECONDARY_BUMP;`.
    3. Если `bestPassed` равен `false`, оценивать все остальные домены по их базовым порогам без прибавки: `const threshold = s.threshold;`.
  - **Критерии приемки:** Чанки, отклоненные «жадным» доменом, попадают в другие подходящие домены. Домены «Дети», «Родители», «Коллеги» наполняются корректно.

- [ ] **PROF-9: Обновление центроидов при инкрементальном импорте заметок** - пересчитывать вектор центроида фасета при добавлении новых чанков
  - **Файл:** `src/features/ai/services/AIProfileFacetService.ts`
  - **Логика:** Метод `incrementalUpdate(noteId)` (строки 374-419) добавляет `noteId` в массив фасета, но не изменяет вектор `f.centroid`. Со временем центроиды устаревают.
  - **Решение:**
    При сопоставлении нового чанка `ch` с фасетом `f` обновлять вектор `f.centroid` с использованием скольжения или взвешенного среднего (например, `0.9 * f.centroid + 0.1 * ch.vector`), а затем заново L2-нормализовать результирующий вектор:
    ```typescript
    if (bestFacet.centroid && bestFacet.centroid.length > 0) {
      const blended = bestFacet.centroid.map((x, idx) => x * 0.9 + ch.vector[idx]! * 0.1);
      let mag = 0;
      for (const val of blended) mag += val * val;
      mag = Math.sqrt(mag);
      bestFacet.centroid = mag === 0 ? blended : blended.map(val => val / mag);
    } else {
      bestFacet.centroid = ch.vector.slice();
    }
    ```
  - **Критерии приемки:** После добавления новой заметки с новыми чанками векторы центроидов измененных фасетов пересчитываются и нормализуются.

- [ ] **PROF-10: Интеграция Reranking в поиск ИИ-психолога** - использовать `AIService.rerank` для сокращения и ранжирования контекста
  - **Файл:** `src/features/ai/hooks/useAIChat.ts`
  - **Логика:** В функции `sendMessage` при обнаружении поискового интента (строки 411-461) выполняется локальный сбор кандидатов-заметок `allNotes`. Сейчас мы берем до 25 заметок напрямую, сжимая их до 2000 символов и формируя гигантский блок контекста.
  - **Решение:**
    1. Если количество кандидатов в `allNotes` больше 5, подготовить список `candidates: { documentId: string; card: string }[]`, где `card` — это `"${note.title}\n${note.content.slice(0, 400)}"`.
    2. Вызвать `const rerankRes = await AIService.rerank({ query: text, candidates });`.
    3. Если `rerankRes.ok`, переупорядочить `allNotes` в соответствии с массивом `rerankRes.documentIds` и оставить только первые 5 наиболее релевантных заметок.
    4. При ошибке или тайм-ауте ранжирования делать graceful fallback на исходный список.
  - **Критерии приемки:** Облачный `rerankNotes` вызывается при поиске в чате, размер контекста уменьшается, ассистент отвечает точнее по смыслу запроса.
