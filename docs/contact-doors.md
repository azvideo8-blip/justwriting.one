# «Три двери контакта» — анализ заметок (тикет для исполнителя / opencode)

Отдельный самодостаточный тикет (намеренно вне других docs, чтобы не конфликтовать с текущими правками). Префикс задач — `DOORS-`.

## Идея (и важные ограничения)
Анализировать, **куда направлено внимание** пользователя в заметках по трём «дверям контакта» (Уэйр): **мысли / чувства / поведение**. Показывать распределение как зеркало для самопознания и мягко подсвечивать «тонкую» дверь.

**КРИТИЧЕСКИ важные границы (не нарушать):**
- Это **измерение/описание, НЕ диагноз.** Никакого типирования личностных адаптаций (трудоголик/скептик/энтузиаст/манипулятор/сопротивленец/мечтатель), никаких имён адаптаций в продукте. По одному наблюдаемому сигналу адаптацию определить нельзя (разные адаптации делят одну открытую дверь) — ИИ ошибётся, поэтому он её и НЕ выводит.
- Формулировка для пользователя — «куда идёт твоё внимание», а не «ты такой-то».
- В чате — только **мягкое приглашение-гипотеза** к недостающей двери, от которого можно отказаться. НЕ директива, НЕ «целевое» терапевтическое вмешательство (мы не знаем, целевая это дверь или защитная «ловушка»). Уже согласуется с `REFLECTION_GUIDE` (со-интерпретация, агентность, без советов).
- **Лицензия:** трихотомия «мысли/чувства/поведение» — общая ТА/КПТ-рамка, использовать можно. Именованная типология адаптаций (Джойнс/Стюарт) — НЕ включать без прав; если позже появится — только как опциональный лицензированный самоопросник по явному opt-in.

## Контекст стека
- Заметки: текст последней версии в IndexedDB (`versions` по `by-document`); саммари — `AIDocumentSummary` (`aiSummaries`).
- Профиль-вкладка: Диагностика → «Профиль AI» (`DiagnosticsPage`, рядом с `ProfileFacets`/`EmbeddingDiagnostics`).
- Чат-контекст: `src/features/ai/hooks/useAIChat.ts` (`sendMessage`) собирает блок контекста (фасеты/заметки) и шлёт в `documentContent`/system.
- Персона-промпты: `functions/src/shared/prompts.ts` + `src/shared/ai/prompts.ts` (`REFLECTION_GUIDE` уже есть).
- Гейт: `tsc` (root+functions), eslint, build. Без серверных вызовов v1 — без лимитов.
- ⚠️ `useAIChat.ts` и `DiagnosticsPage.tsx` сейчас активно правятся — встраиваться аккуратно, минимальными вставками.

## DOORS-1 — Локальный лексический анализатор (без ИИ, без лимитов)
**Файл (новый):** `src/features/ai/utils/contactDoors.ts`
**Задачи:**
- `analyzeDoors(text: string): { thinking: number; feeling: number; behavior: number }` — доли (сумма = 1) по совпадению маркеров; если маркеров нет — вернуть нули/равномерно, пометить «мало данных».
- Русские словари маркеров (по основам/подстрокам, с лёгким учётом отрицания «не …» — оно не меняет дверь, «не злюсь» всё равно про чувства):
  - **мысли:** думаю, мысль, кажется, считаю, понимаю, осознаю, анализ, потому что, вывод, наверное, логично, размышля, идея, план(ирую).
  - **чувства:** чувству, ощущаю, злюсь/злость, боюсь/страх, тревог, грустно/грусть, радост, обиincome(обида), стыд, вина, раздраж, тепло, спокойн, устал (как состояние).
  - **поведение:** сделал, не сделал, пошёл, поехал, написал, начал, бросил, купил, встал, лёг, позвонил, провёл, занимался, работал.
  - (Списки — стартовые, расширяемые; держать как константы в файле.)
- `aggregateDoors(perNote: {doors, ts}[])` → общий профиль + динамика (доли по неделям/месяцам по `ts`).
- Юнит-тест: текст с явным перекосом в одну дверь → она доминирует; пустой → «мало данных».
**Приёмка:** на реальной заметке распределение правдоподобно; считается мгновенно, без сетевых вызовов.

## DOORS-2 — Виджет «Три двери контакта» в профиле
**Файлы:** новый компонент `src/features/ai/components/ContactDoors.tsx`, врезка в `DiagnosticsPage` (вкладка «Профиль AI», рядом с `ProfileFacets`).
**Задачи:**
- Загрузить заметки (текст последней версии + дата) локально, прогнать `analyzeDoors`, показать:
  - распределение «Мысли / Чувства / Поведение» (бар или донат, %),
  - подпись-зеркало: «Сейчас твоё внимание чаще идёт через **мысли**; тоньше всего — **чувства**»,
  - **динамику во времени** (как менялось),
  - дисклеймер: «Это про то, куда направлено внимание в записях, а не диагноз».
- Кнопка «Обновить». Кэшировать результат в рамках сессии.
**Приёмка:** видно распределение + тренд + аккуратная формулировка без типирования.

## DOORS-3 — Мягкий сигнал в чат (приглашение к тонкой двери)
**Файл:** `src/features/ai/hooks/useAIChat.ts` (минимальная вставка в сборку контекста).
**Задачи:**
- Посчитать агрегат дверей (по последним N заметкам) — переиспользовать `analyzeDoors`; кэшировать.
- Добавить в контекст для психо-персон одну строку-подсказку (в system/контекст-блок, НЕ как директиву пользователю), например: «Наблюдение: в записях пользователь чаще опирается на мысли; чувства звучат реже. Если уместно — мягко, гипотезой пригласи заметить чувства под мыслями; не дави, дай возможность отказаться.»
- Не на каждое сообщение — только когда уже подмешивается контекст/фасеты (не плодить лишнего).
**Приёмка:** персона иногда уместно и бережно приглашает к недостающей двери вопросом, без диагнозов и директив.

## DOORS-4 (опц.) — «Зеркало + спроси» (пользователь подтверждает сам)
**Идея:** вместо вывода адаптации — показать зеркало и спросить «больше пишешь о мыслях, чем о чувствах — про тебя?». Ответ пользователя сохранить как его собственную пометку (не AI-диагноз).
**Задачи:** в `ContactDoors.tsx` — опциональный вопрос + сохранение ответа (локально, напр. в профиле). Использовать ответ как лёгкий ориентир (приоритетнее, чем авто-оценка).
**Приёмка:** пользователь может подтвердить/опровергнуть наблюдение; его ответ учитывается.

## Порядок
DOORS-1 (анализатор) → DOORS-2 (виджет) → DOORS-3 (мягкий сигнал в чат) → DOORS-4 (зеркало+спроси) → DOORS-5 (ИИ-уточнение, позже).
v1 = DOORS-1..3: полностью локально, без ИИ-вызовов и лимитов, без типирования.
