# Улучшения от аудита (Antigravity) — что делать, что не трогать

Самодостаточный файл. Префикс — `IMPR-`. Источник: список из 10 улучшений от антигравити-аудита; часть уже реализована в коде на момент аудита (аудит их не увидел), часть — реальные гэпы. Ниже — только то, что решено внедрить (высокий + низкий приоритет). Три пункта из исходного списка явно исключены (см. раздел «Не делать» в конце) — не начинать их без отдельного запроса.

---

## IMPR-1 — Warning при закрытии таба с несинхронизированными правками 🟠
**Контекст (уже есть, не путать):** `beforeunload`/`pagehide` уже подключены в `src/features/writing/hooks/useDraftCore.ts` (`useSyncUnloadSave`, строки ~106-144) — но они тихо флашат черновик в `localStorage`, без `preventDefault()` и без браузерного диалога подтверждения. Это не то же самое, что просить пользователя подтвердить уход при незалитой синхронизации.
**Задачи:**
1. Использовать существующий sync-статус (см. `sync-resilience.md` SYNC-3, `SyncService.getPendingCount()` / `syncQueue` в `src/core/storage/localDb.ts`) — если `pendingCount > 0` и `auto_sync_enabled !== 'false'`, добавить отдельный `beforeunload`-листенер (не трогать существующий `useSyncUnloadSave`), который вызывает `e.preventDefault()` (и `e.returnValue = ''` для старых браузеров).
3. Не блокировать уход, если пользователь гость без активной синхронизации или очередь пуста.
**Приёмка:** закрытие/обновление таба при непустой `syncQueue` показывает стандартный браузерный диалог подтверждения; при пустой очереди — уходит без диалога, как сейчас.

## IMPR-2 — Fuzzy-search в архиве 🟡
**Контекст:** сейчас в `src/features/archive/hooks/useArchiveSearch.ts` (строки 12-13) — прямой `.toLowerCase().includes()` по `content`/`title`. Опечатки, разная раскладка, окончания слов — не находятся.
**Задачи:**
1. Добавить зависимость `fuse.js`.
2. В `useArchiveSearch.ts` — при непустом `debouncedSearchQuery` собирать `Fuse` по `title`, `content`, `tags`; порог `threshold` в районе 0.3–0.4 (не давать слишком много шума).
3. Пересчитывать индекс `Fuse` только когда меняется список сессий (мемоизация), не на каждый keystroke.
**Приёмка:** поиск с опечаткой в одну-две буквы (напр. «кнцепция» вместо «концепция») находит документы с правильным написанием; точные совпадения продолжают находиться как раньше.

## IMPR-3 — Экспорт транскрипта AI-диалога 🟡
**Контекст:** `src/features/ai/pages/AIPage.tsx` — есть чат с ИИ-персонажами, нет кнопки экспорта. Пользователи теряют идеи после закрытия диалога.
**Задачи:**
1. Кнопка «Экспорт» в шапке диалога на `AIPage.tsx`.
2. Формат Markdown: заголовок с датой, роли жирным (`**Пользователь**`, `**<имя персонажа>**`), сообщения по порядку.
3. Скачивание через `Blob` + `text/markdown`, имя файла — по названию диалога/дате.
**Приёмка:** клик по «Экспорт» скачивает `.md`-файл с полной историей диалога в читаемом виде.

## IMPR-4 — Экспорт диагностического отчёта 🟡
**Контекст:** `src/features/ai/pages/DiagnosticsPage.tsx` уже собирает много данных (sync-статус через `SyncDiagnostics`, embedding-диагностику и т.д.), но всё это видно только на экране — не скопировать/не отправить в саппорт.
**Задачи:**
1. Кнопка «Скачать диагностику» на `DiagnosticsPage.tsx`.
2. Собрать в один JSON: `navigator.userAgent`, locale, версия приложения (см. `CHANGELOG.md`/константу версии, если есть), `syncQueue.length`, статус подключения Firestore (`isFirestoreConnected`), базовые счётчики из локальной БД (кол-во сессий/документов), без содержимого заметок и без секретов/токенов.
3. Скачивание как `justwriting-diagnostics.json`.
**Приёмка:** клик скачивает JSON с метаданными системы, без пользовательского текста и без учётных данных.

## IMPR-5 — Типографика и layout в настройках редактора 🟡 (низкий приоритет)
**Контекст:** `src/features/settings/components/EditorTab.tsx` — сейчас без контролов line-height/font-size/max-width/font-family.
**Задачи:**
1. Добавить состояния в существующий контекст настроек письма (`useWritingSettings` или аналог — найти актуальное имя перед началом, могло измениться).
2. Рендер контролов в `EditorTab.tsx`: font-size, line-height, ширина контейнера (narrow/medium/wide), выбор шрифта из короткого curated списка (Inter, Lora, Playfair Display, EB Garamond, JetBrains Mono — подключить как Google Fonts, проверить лицензии/оффлайн-доступность с учётом уже подключенного Service Worker).
3. Применить в обёртке редактора (`WritingEditor.tsx` или актуальный компонент).
**Приёмка:** изменение любого параметра в Настройках сразу видно в активном окне письма.

## IMPR-6 — Импорт документов (.md/.txt/.docx) 🟡 (низкий приоритет)
**Контекст:** сейчас вставка текста в редактор — только copy-paste.
**Задачи:**
1. Drop-zone на экране начала письма (drag&drop).
2. `.md`/`.txt` — читать как plain text. `.docx` — через `mammoth.js` (конвертация в markdown/plain text).
3. Имя файла (без расширения) — как заголовок сессии по умолчанию, если пользователь ещё не задал свой.
**Приёмка:** drag&drop `.md`/`.txt`/`.docx`-файла на экран письма подставляет содержимое в редактор и заголовок из имени файла.

## IMPR-7 — Heatmap активности в профиле 🟡 (низкий приоритет)
**Контекст:** `src/features/profile/components/ProfileHero.tsx` / `Achievements.tsx` — есть стрики (`StreakDots.tsx`), нет годовой карты активности.
**Задачи:**

2. Агрегация количества слов по дням за последний год из существующих сессий/локальной БД (уточнить источник — сессии в `localDb`, не Firestore, чтобы не грузить сеть).

**Приёмка:** в профиле рендерится грид с корректной интенсивностью по историческим данным письма.

## Не делать (явно исключено из этого списка)

- **PWA через `vite-plugin-pwa`** — уже есть рабочий кастомный Service Worker (`public/sw.js`, `CACHE_VERSION`, nav-cache, network-first) и `public/manifest.json` с shortcuts. Замена на автогенерируемый workbox — риск регресса: уже было продовое инцидент со stale IndexedDB после смены версии кэша при ручном SW. Не трогать без отдельного тикета и явного согласия.
- **Кастомный undo/redo стек для редактора** — не заводить. Риск сломать нативный undo в contenteditable выше пользы без подтверждённой жалобы. Если появится конкретный баг с потерей undo — отдельный тикет с репро.
- **Локальное шифрование БД (AES-GCM/PBKDF2)** — уже полностью в проде: `src/core/crypto/encrypt.ts` (PBKDF2 300k итераций, AES-KW обёртка ключа, AES-GCM контента), `EncryptionService.ts`, `EncryptionMetaService.ts`, UI в `EncryptionPasswordModal.tsx` / `AccountVaultSection.tsx`, миграция legacy-ключей в `LegacyKeyMigration.ts`. Дублировать не нужно — это не гэп, а то, что аудит не заметил.
