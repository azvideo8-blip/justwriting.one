# Диалог с ИИ «до идеала» — тикеты (для исполнителя / opencode)

Самодостаточный план. Контекст: личный дневник на русском (практика «аскеза» — ежедневные записи) + ИИ-персоны (психолог-панель `group_psychology`, `cbt`, `coach`, `editor`, `custom`), которые рефлексируют ВМЕСТЕ с пользователем, опираясь на его заметки (chunked-эмбеддинги, гибридный поиск, фасеты-профиль). Цель — сделать диалог непрерывным (память), эмоционально точным, грунтованным и проверяемым. Основано на ресёрче (источники в конце).

Стек: React+TS+Vite; Firebase Functions; Vercel. ИИ — OpenRouter (чат `deepseek/deepseek-v4-flash`, reasoning-модель), эмбеддинги `qwen/qwen3-embedding-8b`, служебная модель `AI_FACET_MODEL` (env, по умолч. `openai/gpt-oss-20b:free`).

## 0. Контекст и грабли (читать перед всем)

**Что уже есть:** chunked-эмбеддинги (схема v3, тексты чанков), гибридный поиск (вектор+BM25 RRF + cloud-рерэнк + поиск по именам), фасеты профиля (домены жизни + люди), `NOTES_GUARD`, персона-промпты, инкрементальная индексация. Ретрив — на уровне best-practice.

**Чего НЕ хватает (отсюда тикеты):** памяти МЕЖДУ диалогами; рефлексивного крафта в промптах; CBT/MI-структуры; темпоральных наблюдений; цитат/проверяемости; измерения качества.

**Ключевые файлы:**
- Персона-промпты — **ДВА файла, держать синхронными:** `functions/src/shared/prompts.ts` и `src/shared/ai/prompts.ts` (`PERSONA_PROMPTS`, `TOPIC_GUARD`, `NOTES_GUARD`). Персоны: `group_psychology`, `cbt`, `coach`, `editor`, `custom`.
- Оркестрация чата — `src/features/ai/hooks/useAIChat.ts` (`sendMessage`).
- Бэкенды чата — **ДВА, держать синхронными:** `functions/src/ai/chatWithAI.ts` (callable) и `api/chat.ts` (стрим). Оба собирают system-промпт и вставляют `documentContent`.
- Диалоги — `src/features/ai/services/AIDialogueService.ts`, стор `aiDialogues` (`AIDialogue { id, title, personaId, messages[], createdAt, updatedAt, responseLength? }`).
- Саммари заметок — `AISummaryService`; фасеты — `AIProfileFacetService` (есть `firstAt/lastAt/centroid/noteIds`, тренды).
- Эталон служебного callable (лимит-экемпт, дешёвая модель, парс/реджект) — `functions/src/ai/summarizeFacet.ts` (модель через `AI_FACET_MODEL`, оверрайд `model` в `generate`).
- IndexedDB — `src/core/storage/localDb.ts`: `openDB('justwriting-local', 7, …)`, апгрейды по `if (oldVersion < N)`. Шифрование облака — `maybeEncrypt/maybeDecrypt` (см. `AIEmbeddingService.ts`).

**Грабли:**
- Новые служебные LLM-вызовы (извлечение памяти, цитаты-чек и т.п.) — **только** `withinGlobalDailyLimit` + `recordUsage`, БЕЗ `checkDailyLimit`/`checkRateLimit` (иначе 429 и трата чат-лимита).
- DeepSeek течёт рассуждениями; для структурных задач — `AI_FACET_MODEL`. `sanitizeAiResponse` уже срезает `<reasoning>` (OPT-8).
- Любая правда правится в ОБОИХ prompt-файлах и ОБОИХ бэкендах.
- Новый стор → бамп схемы БД (сейчас 7 → 8) блоком `if (oldVersion < 8)`. Не трогать существующие блоки.
- Гейт перед коммитом: `npx tsc --noEmit` (root + `functions`), `npx eslint <файлы>`, `npm run build`. Деплой функций: `cd functions && firebase deploy --only functions:<name>`; новая v2-callable → проверить `curl -X POST <url> -d '{"data":{}}'` = 401.
- Персоны должны остаться РАЗНЫМИ по характеру; общие принципы добавляем, не унифицируя голоса.

## DLG-1 — Память между диалогами (Mem0-стиль) 🔴 крупный, наибольший эффект
**Проблема:** каждый диалог изолирован — персона не помнит, что обсуждали в прошлых беседах («на прошлой неделе разбирали вину из-за денег»). Память есть только о заметках, не о разговорах.
**Подход (паттерн Mem0):** после диалога LLM извлекает устойчивые единицы памяти; они хранятся и подмешиваются в будущие чаты по релевантности.
**Задачи:**
1. **Стор** `aiChatMemory` (БД v8): `{ id, kind: 'fact'|'insight'|'commitment'|'preference', text, sourceDialogueId, createdAt, updatedAt, vector?: number[] }`. Локально + (опц.) зашифрованно в облако, как эмбеддинги.
2. **Callable** `extractChatMemory` (лимит-экемпт, `AI_FACET_MODEL`, формат как `summarizeFacet`): вход — сообщения диалога; выход — список кратких единиц памяти (факты о жизни, инсайты, договорённости/намерения, предпочтения общения). Реджект рассуждений.
3. **Сервис** `AIChatMemoryService`: `extractFromDialogue(dialogueId)` (зовёт callable, эмбеддит текст каждой единицы через `AIService.embed`, сохраняет), `getRelevant(queryVector, k)` (косинус), `getAll`, дедуп (v1 — ADD + простое слияние почти-дублей по косинусу; UPDATE/DELETE — позже).
4. **Триггер:** вызывать `extractFromDialogue` фоном после диалога (по завершению/смене диалога в `useAIChat`, либо после каждых N обменов), не блокируя UI.
5. **Инжект:** в `useAIChat.sendMessage` достать топ-k релевантных единиц памяти (по эмбеддингу запроса — переиспользовать вектор из OPT-1) и добавить в контекст отдельной секцией «Что ИИ помнит о пользователе из прошлых бесед: …» (через system, см. OPT-5 / DLG-5).
**Приёмка:** в новом диалоге ассистент уместно ссылается на ранее обсуждённое; единицы памяти видны в Диагностике (стор `aiChatMemory`); извлечение не блокирует UI и не жжёт чат-лимит.

## DLG-2 — Переписать промпты персон под рефлексивный диалог 🟠 дёшево, большой «felt» эффект
**Проблема:** ответы получаются консультативно-сухими/общими. Ресёрч компаньонов для рефлексии даёт конкретные принципы.
**Задачи:** в ОБОИХ prompt-файлах добавить общий блок `REFLECTION_GUIDE` (и подмешать его к персонам, сохранив их характеры):
- **Прогрессивные вопросы:** от поверхностного наблюдения вглубь, не сразу совет.
- **Со-интерпретация:** «давай вместе заметим…», не вердикт эксперта.
- **Без общих веллнес-фраз:** только привязка к конкретным паттернам/цитатам из заметок пользователя.
- **Эмоциональная подстройка:** подхватывать тон и интенсивность; валидировать противоречия, не упрощать; **никакого пустого подбадривания**.
- **Агентность пользователя:** инсайт как топливо для ЕГО решения, не директива; прозрачность про «чего не знаю».
- Открытые вопросы («что для тебя здесь главное?»), любопытная позиция, привязка к ценностям.
**Приёмка:** ответы стали конкретнее (ссылаются на детали из заметок), задают вглубь, не подбадривают вхолостую; персоны по-прежнему различимы по голосу.

## DLG-3 — Структура CBT / мотивационного интервью для соответствующих персон 🟠
**Проблема:** для `cbt`/`coach`/`group_psychology` нет терапевтического каркаса; модель «ловит» неадаптивные мысли лучше, чем переформулирует (вывод из ресёрча).
**Задачи (промпт-уровень, без жёсткого стейт-машина в v1):**
- **`cbt`:** зашить лёгкий флоу когнитивной реструктуризации — ситуация → автоматическая мысль → когнитивное искажение → бережный сократический рефрейм. Не выдавать готовый вывод — вести вопросами.
- **`coach`/`group_psychology`:** техника мотивационного интервью — «сложные отражения» (перефраз + добавленный смысл), вызвать собственную мотивацию пользователя, не давить.
- Опц. (v2): хранить «шаг» сценария в диалоге для последовательности.
**Приёмка:** CBT-персона ведёт по шагам реструктуризации; коуч даёт отражения, а не советы; не звучит как чек-лист.

## DLG-4 — Темпоральные наблюдения и проактивная рефлексия 🟡
**Проблема:** не используется динамика во времени, хотя данные есть (`facet.firstAt/lastAt`, тренды, даты заметок).
**Задачи:**
- В блок фасетов (`useAIChat`) добавлять темпоральные пометки: «тема активна сейчас / затихла с <дата> / новая». 
- Изредка — **проактивный** рефлексивный вопрос, привязанный к паттерну («месяц назад ты часто писал про усталость — как сейчас?»). «Что-если» подавать как гипотезу, помеченную как гипотеза.
- Не спамить: проактив — например, при открытии диалога или раз в N обменов.
**Приёмка:** ассистент уместно отмечает изменения во времени и иногда задаёт проактивный вопрос по реальному паттерну.

## DLG-5 — Грунтованность: контекст в system + цитаты 🟡
**Проблема:** контекст идёт фейковым user-ходом (см. OPT-5); нет проверяемых ссылок на источники; faithfulness ≠ correctness (модель может пост-рационализировать).
**Задачи:**
- Контекст (заметки/фасеты/память) — в **system-промпт** размеченными секциями, не как user-ход (синхронно с OPT-5).
- **Цитаты:** просить модель помечать, на какие заметки (название/дата) она опирается; в UI чата рендерить эти ссылки кликабельно (открыть заметку) — совпадает с «доказательностью» фасетов.
- Опц.: лёгкая self-/NLI-проверка ответа против контекста для флага неподтверждённых утверждений.
**Приёмка:** в `messages` нет синтетического user-документа; ответ содержит проверяемые ссылки на заметки; пользователь может открыть источник.

## DLG-6 — Контур оценки качества (как измерять «идеал») 🟢 опционально, но важно
**Проблема:** «идеал» без метрик недостижим; ресёрч 2025 делает упор на faithfulness-оценку (FaithJudge / LLM-as-judge).
**Задачи:**
- Набор проб (10–20 типовых запросов к дневнику) + скрипт/панель в Диагностике: гоняет диалог, оценивает грунтованность (LLM-as-judge: «подтверждается ли ответ приведённым контекстом?»), считает долю галлюцинаций и «попадание в тему».
- Прогонять до/после изменений промптов и ретрива.
**Приёмка:** есть воспроизводимая оценка faithfulness/релевантности; видно, улучшает ли изменение качество.

## Порядок
DLG-2 (промпты — дёшево, сразу заметно) → DLG-1 (память — главный пробел) → DLG-4 (темпоральность) → DLG-3 (CBT/MI) → DLG-5 (контекст в system + цитаты; синхронно с OPT-5) → DLG-6 (оценка).
Самые сильные рычаги «до идеала»: **DLG-1 (непрерывность)** и **DLG-2 (живой, не сухой разговор)**.

## Открытые решения
- Память: дедуп v1 (ADD + слияние) vs полноценный ADD/UPDATE/DELETE (Mem0) — начать с простого.
- Память локально vs зашифрованный облачный синк — по аналогии с эмбеддингами/саммари.
- Триггер извлечения памяти: по завершению диалога vs каждые N обменов — выбрать по UX.
- Согласовать DLG-5 с OPT-5 из `docs/optimising_ai.md` (контекст в system) — делать вместе.
