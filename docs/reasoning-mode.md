# Режим ответа «С рассуждениями» — тикет (для исполнителя / opencode)

Отдельный самодостаточный тикет (намеренно вне `ai-dialogue-ideal.md`, чтобы не конфликтовать с текущими правками). Зависимостей от других тикетов нет, но согласуется с OPT-5 (контекст в system) и OPT-8 (стрип reasoning).

## Идея
Дать пользователю режим, где видно **ход мысли** ИИ (`<reasoning>`: анализ записи, выбор «голосов», синтез) перед **ответом** (`<answer>`) — как extended-thinking у Claude/o1. Это фича промпта + парсинга + UI. **Отдельная модель НЕ обязательна** (текущая `deepseek-v4-flash` — reasoning-модель и уже думает); для бóльшей глубины — `deepseek-v4-pro`.

## Контекст (важно)
- Персона-промпты УЖЕ построены как `//<reasoning> … </reasoning>` + `<answer> … </answer>` (см. `functions/src/shared/prompts.ts` и `src/shared/ai/prompts.ts`, `PERSONA_PROMPTS`).
- Но `sanitizeAiResponse` (`functions/src/shared/aiUtils.ts`, OPT-8) сейчас **вырезает** `<reasoning>…</reasoning>` и `//<reasoning>`. Поэтому ход мысли генерируется и выбрасывается. Режим должен НЕ резать reasoning, а показать его.
- Длина ответа уже есть как `responseLength: 'short' | 'standard' | 'detailed'` — прокинута через `useAIChat` → `chatWithAI.ts` (callable) + `api/chat.ts` (стрим) → системный промпт. Это та же ось селектора, куда добавляется новый вариант.
- **Два бэкенда** (`chatWithAI.ts` callable и `api/chat.ts` стрим) и **два prompt-файла** — править синхронно.
- Грабли: служебные LLM-вызовы — только `withinGlobalDailyLimit`; оверрайд модели — параметр `model` в `generate` (`functions/src/shared/aiProvider.ts`, уже есть, см. `summarizeFacet`/`AI_FACET_MODEL`). Гейт: `tsc` (root+functions), eslint, build. Деплой функций: `cd functions && firebase deploy --only functions:chatWithAI`.

## Задачи
1. **Режим:** добавить вариант `reasoning` в селектор длины ответа (значение `responseLength = 'reasoning'`, или отдельный флаг `showReasoning` — на выбор). Подпись в UI: «С рассуждениями». Подразумевает «детально» + показ хода мысли. Прокинуть через `useAIChat` → оба бэкенда.
2. **Промпт:** в режиме reasoning гарантировать формат вывода: сначала `<reasoning>…развёрнутый анализ и план…</reasoning>`, затем `<answer>…глубокий структурированный разбор…</answer>`. Психо-персоны (`group_psychology`/`cbt`/`coach`) уже так; для `editor`/`custom` добавить инструкцию в этом режиме.
3. **Backend — sanitize по режиму:** `sanitizeAiResponse` сейчас всегда удаляет reasoning. Сделать его **mode-aware** (передать флаг): в режиме reasoning НЕ удалять reasoning, вернуть обе части — например `{ reasoning, answer }`, либо размеченный текст. HTML по-прежнему чистить через DOMPurify (без удаления текста внутри `<reasoning>`/`<answer>`). Обычные режимы — поведение без изменений (reasoning режется).
4. **Стриминг (`api/chat.ts`):** reasoning идёт первым, ответ — потом. Парсить поток на клиенте: пока не встретился `</reasoning>` — копить в «ход мысли»; после — остальное стримить как `<answer>`. UI показывает «🧠 Ход мысли» в реальном времени (сворачиваемо), затем основной ответ.
5. **Модель:** для режима reasoning роутить на `deepseek-v4-pro` через `model`-оверрайд в `generate` (опц. env `AI_REASONING_MODEL`). Учесть, что это дороже/медленнее — ок для премиум-режима.
6. **UI:** блок «Ход мысли» свёрнут по умолчанию, раскрывается по клику; основной ответ виден сразу. Решить, сохранять ли reasoning в истории диалога (свёрнуто) или показывать только в живой сессии.
7. (Опц., большой заход) Если нужна именно Opus-глубина как в примере-эталоне — отдельный провайдер **Anthropic/Claude** в `aiProvider` (новый ключ/SDK), только для этого режима.

## Приёмка
- В режиме «С рассуждениями» пользователь видит сворачиваемый ход мысли (стримится в реальном времени) + глубокий ответ.
- В обычных режимах (`short`/`standard`/`detailed`) reasoning по-прежнему скрыт.
- На `deepseek-v4-pro` глубина разбора заметно выше, чем на `flash`.
- `tsc` 0 (root+functions), eslint 0, build зелёный; оба бэкенда синхронны.
