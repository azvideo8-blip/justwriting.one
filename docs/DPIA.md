# DPIA — Data Protection Impact Assessment

## justwriting AI Features

**Дата:** 18 июня 2026  
**Версия:** 1.0  
**Ответственный:** разработчик justwriting

---

## 1. Описание обработки данных

### 1.1 ИИ-персоны (CBT, групповая психология, коуч, редактор)

**Что обрабатывается:** текст пользовательских заметок (до 50K символов за запрос) передаётся в сторонний ИИ-API (Google Gemini, OpenRouter) для генерации ответов в рамках выбранной персоны.

**Цель:** предоставление ИИ-ассистента для рефлексии, когнитивного реструктурирования (CBT), коучинга и редактирования текстов.

**Категории данных:** личные тексты пользователя, которые могут содержать информацию о психоэмоциональном состоянии, отношениях, здоровье, травматическом опыте.

### 1.2 AI-саммаризация

**Что обрабатывается:** текст заметки анализируется ИИ для извлечения тона, тем, инсайтов, фактов, упомянутых людей (имена + отношения).

**Цель:** автоматическое аннотирование заметок для удобства навигации и поиска.

### 1.3 AI-портрет (психологический профиль)

**Что обрабатывается:** на основе саммари заметок ИИ строит профиль пользователя (до 100K символов) — жизненные домены, темы, динамика.

**Цель:** персонализация ИИ-ответов на основе контекста пользователя.

### 1.4 Отслеживание настроения (mood tracking)

**Что обрабатывается:** пользователь выбирает эмоцию (радость, грусть, злость, страх, отвращение, интерес) после сессии письма.

**Цель:** отслеживание эмоционального состояния во времени.

## 2. Оценка необходимости и пропорциональности

| Критерий | Оценка |
|----------|--------|
| Законное основание | Согласие (Art 6(1)(a) + Art 9(2)(a) для special category) |
| Цель | Вспомогательный инструмент для рефлексии и письма, НЕ медицинская или психологическая помощь |
| Пропорциональность | Текст передаётся во внешний API только при явном действии пользователя (отправка в чат / запрос саммари) |
| Минимизация | Ограничение 50K символов на запрос, 100K на портрет, дневной лимит 5-50 запросов |
| Автоматизация | ИИ-ответы носят рекомендательный характер, не заменяют профессиональную помощь |

---

## 3. Оценка рисков

| Риск | Вероятность | Влияние | Меры снижения |
|------|-------------|---------|---------------|
| Раскрытие чувствительных данных третьим лицам (Gemini/OpenRouter) | Средняя | Высокое | Текст не сохраняется на серверах justwriting после обработки; шифрование AES-256-GCM при хранении; передача только при явном действии пользователя |
| Неправильная интерпретация ИИ-ответа как медицинской рекомендации | Средняя | Высокое | Дисклеймер в Terms of Service; ИИ-персоны не диагностируют; явное указание "не профессиональная консультация" |
| Профилирование пользователя (AI-портрет) без информированного согласия | Низкая | Высокое | Privacy Policy раскрывает портрет; генерация только после согласия с privacy; портрет можно не использовать |
| Утечка психологических данных через аналитику/ошибки | Низкая | Высокое | PII scrubbing в Sentry (hashed UID, no documentId); consent-gated PostHog; maskAllText в Sentry |
| Несовершеннолетние используют ИИ-функции | Низкая | Среднее | Age gate 16+ при регистрации; Terms of Service указывают возраст 16+ |
| Уязвимость данных при хранении в облаке | Низкая | Высокое | AES-256-GCM шифрование; PBKDF2 300K итераций; ключ не хранится на сервере |

---

## 4. Меры безопасности

1. **Шифрование at rest:** AES-256-GCM, ключ выводится из пароля пользователя через PBKDF2 (300K итераций), не хранится на сервере
2. **Шифрование in transit:** HTTPS для всех соединений
3. **Минимизация данных:** дневные лимиты (5-50 запросов), ограничение размера контента
4. **Санитизация ввода/вывода:** 15 паттернов инъекций, DOMPurify на AI output, специальная санитизация токенов
5. **Rate limiting:** per-user daily limit + 10s cooldown + global cap
6. **Consent:** opt-in аналитика, privacy modal при первом входе
7. **PII protection:** hashed UID в Sentry, scrub userId/documentId из error context
8. **Auto-lock:** CryptoKey очищается через 15 мин неактивности
9. **Право на удаление:** Privacy Policy описывает право на удаление данных

---

## 5. Рекомендации

1. **Добавить granular consent для AI-функций** — отдельное согласие на обработку special-category данных (Art 9), помимо общего privacy consent
2. **Оценить EU AI Act compliance** — ИИ-персоны (CBT, психология) могут подпадать под риск-категории AI Act; рассмотреть transparency obligations
3. **Не позиционировать как медицинский инструмент** — ИИ-персоны предоставляют вспомогательную рефлексию, не диагноз или лечение
4. **Рассмотреть DPO** — при масштабировании может потребоваться назначение Data Protection Officer
5. **Регулярный пересмотр** — обновлять DPIA при добавлении новых ИИ-функций или изменении процессоров данных

---

## 6. Заключение

Обработка special-category данных (психоэмоциональный контент) осуществляется на основе явного согласия пользователя (Art 9(2)(a)). Меры безопасности включают шифрование, минимизацию, rate limiting, PII scrubbing. Риски оценены как управляемые при условии реализации рекомендаций. ИИ-функции позиционируются как вспомогательные инструменты для рефлексии и письма, не как медицинские или психологические услуги.
